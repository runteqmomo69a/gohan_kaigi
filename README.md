# gohan_kaigi
卒業制作：お店選び共有アプリ

## サービス概要
複数人で「どこのお店に行くか」を簡単に決められる、提案・投票型のごはん/カフェ選び共有アプリ。  
URL共有でみんなが気になるお店を登録し、いいねで比較することでみんなの「行きたい！」を可視化し最適なお店を選べる。  
幹事の負担を減らしみんなでスムーズに楽しくお店を決める体験を提供する。  

## このサービスへの思い・作りたい理由
前職で複数人のランチや飲み会のお店を決めるとき、幹事役として「全員の意見を集めるのが大変」「決めるまでが時間かかる」というストレスがあった。
また友達とのランチやカフェ選びでも、「どこがいい？」「行きたいお店ある？」というやり取りが長引いたり、
せっかく候補を出してもスクショやURLがトークに埋もれてしまって見返しにくかったり、お店の情報を一目で比較しづらいことで不便さを感じていた。
そんな経験からみんなの意見を簡単に集めて比較できる “ごはん会議アプリ” があれば絶対便利という思いでこのサービスを作ることにした。

## ユーザー層について
- 職場のランチや飲み会でお店を決める人
- 友達・恋人・家族とごはんやカフェに行く人
- カフェ巡りやランチが好きな人

## サービスの利用イメージ
- ユーザーは会員登録・ログインしてイベント（例：金曜ランチ会）を作成。  
- イベントを作ると、参加者用の ユニークURL が発行される。  
- 共有されたURLにアクセスすると、ログイン画面（or 新規登録画面）に誘導され、ログイン後にイベントページへ遷移する。
- ログインしただけではイベントの“閲覧のみ”の状態であり、参加者としては登録されない。
- ログイン後、ユーザーが「このイベントに参加する」ボタンを押すと、イベントの参加者として記録され、投稿・いいね等の操作が可能になる。
- 参加登録されたイベントは、ユーザーの 「参加イベント一覧」 に自動で保存され、過去に参加したイベントページもいつでも再確認できる。
- 参加者はログインした上で、イベントページの「気になるお店を登録」から店名・URL・メモ等を追加。  
- 登録されたお店にはユーザーと紐づくいいねやコメントをつけることができるため誰が提案・投票したかを追跡できる。
- お店一覧は登録順／いいね順で並び替えができ、ランキングとして表示される。

## ユーザーの獲得について
- SNS（X / Instagram）でのシェア
- LINEでURL共有できるため、日常利用の流入が自然に発生

## サービスの差別化ポイント・推しポイント
- 「食べログ」や「Google Map」は「検索」には強いが 複数人で協議・比較に特化した機能を持たない。
- LINEグループでは情報が流れて 比較しにくい。

## 機能候補
### MVPリリース
- 会員登録・ログイン機能
- イベントの作成・編集・削除
- イベントごとのユニークURL発行
- "このイベントに参加する"ボタン
- イベントページごとにお店候補の登録（店名、URL、メモ 等）
- お店へのいいね機能
- お店一覧の並び替え、ランキング表示（登録順 / いいね順）
- 参加イベント一覧・作成イベント一覧の表示

### 本リリース（予定）
- コメント機能
- 写真投稿機能
- カテゴリーやタグ（カフェ / ランチ / 夜ごはん）機能で絞り込み
- Google Map 埋め込み
- 通知機能（いいね・提案が増えたとき）
